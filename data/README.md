# data/ - Raw, Processed, and Enriched Data

Read this file before opening large data files. Most tasks only need one date-specific JSON file.

## Layout

```text
data/
├── raw/
│   ├── YYYY-MM-DD.json
│   └── enriched/
├── processed/
│   ├── YYYY-MM-DD.json
│   └── enriched/
├── managed_reports/
│   └── index.json
├── cookies.json
├── session_info.json
└── README.md
```

## File Roles

- `raw/YYYY-MM-DD.json`: original Dongchedi API scrape output. Generated by `tools/scraper.py`.
- `processed/YYYY-MM-DD.json`: cleaned report-ready structure. Generated by `tools/processor.py`.
- `raw/enriched/`: intermediate enriched HTML/Markdown/JSON artifacts from detail scraping.
- `processed/enriched/YYYY-MM-DD.json`: organized enriched model detail data. Generated by `tools/enrich.py`.
- `managed_reports/index.json`: local report-management index for the workbench. Stores list/filter/archive/follow-up metadata for generated Markdown reports.
- `cookies.json`, `session_info.json`: local login/session artifacts. Treat as secrets and do not commit.

## Current Data Caveat

Some enriched parameter Markdown can be mismatched with the target model. For example, the 2026-07-06 enriched file contains model titles that do not match the parameter body. Reports must use `tools/dynamic_report_generator.py` validation before quoting enriched configuration details.

## Agent Routing

- Need sales, price, rank, body type, energy type: read `processed/YYYY-MM-DD.json`.
- Need to debug scraper output: read `raw/YYYY-MM-DD.json`.
- Need model configuration details: read `processed/enriched/YYYY-MM-DD.json`, but verify trust status before use.
- Need generated prose: read `output/*.md`, not this directory.
- Need report-management metadata: read `managed_reports/index.json`.

## Naming

Date artifacts use `YYYY-MM-DD.json`. Keep JSON UTF-8 encoded.

## Maintenance Notes

- Before editing data conventions or generated fixtures, record the task context in root `progress.md`; after the change, record affected dates, producers, validation commands, and remaining caveats.
- If layout, naming, trust rules, or generated-data expectations change, update this file plus root `AGENTS.md` and `README.md`.
